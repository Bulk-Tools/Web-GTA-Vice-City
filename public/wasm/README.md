# WebAssembly Engine Output

This directory contains the compiled WebAssembly binaries generated from the C++ engine source code.

## Files

- `engine.js` - Emscripten-generated JavaScript glue code
- `engine.wasm` - Compiled WebAssembly binary

## Build Process

These files are automatically generated by the GitHub Actions workflow (`.github/workflows/build-wasm.yml`) whenever changes are pushed to the `engine/` directory.

### Manual Build (Local Development)

If you want to build the WebAssembly module locally:

1. Install Emscripten SDK:
```bash
git clone https://github.com/emscripten-core/emsdk.git
cd emsdk
./emsdk install latest
./emsdk activate latest
source ./emsdk_env.sh
```

2. Build the engine:
```bash
cd engine
mkdir -p build
cd build
emcmake cmake ..
emmake make
```

3. Copy the output files:
```bash
cp engine.js ../../public/wasm/
cp engine.wasm ../../public/wasm/
```

## Integration

The Wasm module is loaded by the React frontend via `src/wasmLoader.js`, which provides:
- `loadWasmEngine()` - Loads the Wasm module
- `initializeEngine()` - Initializes the C++ engine
- `renderFrame(timestamp)` - Renders a frame
- `shutdownEngine()` - Cleans up resources

## Note

The compiled Wasm files are tracked in git to ensure the frontend always has a working engine binary. They are automatically updated by GitHub Actions on push to main.
